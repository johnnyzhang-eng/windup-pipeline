## 一、当前工作流全貌（最新）

```
[输入] 角色参考图（可正面/任意视角/任意风格）
   │
   ▼ ① 视角规整 ── AI 把角色转成 side-view 侧视（朝右）★新增，为非侧视角色
   │              约束：no shadow · 品红背景(避开角色配色) · 保身份/风格
   ▼ ② 角色卡锁定 ── 基准帧 + 结构化描述（配色/装备/比例）
   │
   ▼ ③ 骨架序列 ── 程序生成连续走路循环骨架（侧视, 8帧, 大跨步, 左右腿区分）
   │
   ▼ ④ 逐帧生成 ── Gemini flash-image, 参考图+骨架条件, 固定描述
   │              约束：no shadow · 品红背景 · 同尺寸同站位
   ▼ ⑤ 抠图去背 ── rembg(u2net) AI 主体抠图（不按颜色抠）
   │
   ▼ ⑥ 逐帧对齐 ── Pillow 按脚底线+躯干中心锚点对齐, 消画布漂移
   │
   ▼ ⑦ 打包 ── sprite sheet(降到256) + JSON/plist + GIF预览
   │
   ▼ ⑧ 引擎预览 ── Phaser(localhost) 播放 / Cocos(MCP,待装) 导入
   [输出] 可进引擎的走路动作包
```

**近期变更**：③④⑤⑥ 已跑通验证；① 视角规整为本次新增（应对非侧视角色如 Lirael）。

## 二、测试日志（时间倒序，最新在上）

<!-- 每条格式：日期 · 测了什么 · 输入 · 结果 · 流程是否因此改变 · 试错标记 -->

### T-01 · Lirael 换角色 + 视角规整（2026-07-13）
- **测什么**：换全新角色（Lirael：正面、像素风、绿袍德鲁伊），验证工作流通用性 + 新增"正面→侧视"步。
- **输入**：`Lirael_character.png`（1024², 正面, 像素画）。
- **采纳的新约束**：prompt 加 `no shadow`；背景用**品红（非绿）**——因角色是绿袍，绿背景会撞色被抠穿（同学建议绿幕，但对绿角色反而错）。
- **结果**：✅ 生成 3 张、成 2 张（`lirael_side_1/3.png`）。身份保住（红发/鹿角/绿兜帽/法杖蓝球/箭袋/符文衣）、像素风在、品红底干净、无阴影。
- **关键发现（视角）**：**"侧面"分正侧面 vs 伪侧面(3/4)**。候选1=伪侧面（微转身、看得到胸前和脸，**细节角色更好看**）；候选3≈正侧面（规整但扁平死板）。→ **对精细角色应默认伪侧面**。
- **连带取舍**：伪侧面**左右镜像会换手/透视反**（法杖会跑错手），左右方向可能需分别生成、不能简单镜像；正侧面才好镜像。骷髅（正侧面）没这问题，换精细角色才暴露。
- **流程变化**：① 新增"视角规整"步；② 视角默认从"正侧面"改为"**伪侧面(3/4)**"（好看优先）；③ 背景选色规则明确"避开角色主色"。
- **试错标记**：⚠️ 同学"绿背景"建议对绿角色是坑，已改品红。

## 三、已固化的踩坑结论（从试错里沉淀，详见 CLAUDE.md）

| 坑 | 结论 | 何时踩的 |
|---|---|---|
| 本机跑 SD1.5 | 11分/帧、质量差 → 弃，丝滑要上云 GPU | SD 试验 |
| 按颜色抠图 | 骨白/绿袍撞背景色被抠穿 → 必须用 rembg AI 抠图 | 骷髅抠图 |
| 背景选色 | 要和角色配色远；绿角色不能用绿背景 | Lirael 分析 |
| 脚下阴影 | 抠不干净 → prompt 加 no shadow 从源头掐 | 骷髅走路 |
| 画布漂移 | 每帧位置不同 → 必须按脚底+躯干中心对齐 | 骷髅走路 |
| 一次出 sheet vs 逐帧 | sheet 帧间"像"不"连"；逐帧+骨架才连贯且保单帧可编辑 | 走路平滑 |
| 动画平滑度 | 本期到原型档即可，非护城河，丝滑留 P1 | 全程 |
| file:// 预览 | 空白假死 → 必须 http.server | Phaser 预览 |
| Cocos | GUI 需登录，AI 装不了 → MCP 驱动 + 人工最后一步 | Cocos 调研 |
| 后台进程 | 用完必杀，曾空转 56% CPU 发热 | 发热排查 |
